Nest介绍

发表于 2018-03-13 | 分类于 Node.js ， Nest |

Nest是一个为了解决Node.js服务器端应用架构问题而产生的框架，语法和Angular很像，可以使用TypeScript开发。它是基于Express的。

之前一直使用Express开发，开发出来的应用也能满足项目的需求，但在之后的维护、增加功能的过程中，却越来越难。虽然在使用Express开发的过程中也有意识的对代码结构做了一些约定，但还是不尽人意。从这方面来说，Nest确实是我想要的Node.js框架。

Nest提供了一些基本的构造块：Module、Controller、Component、Middleware、Pipe、Exception Filter、Guard、Interceptor。官网提供了一个开始程序，只有一个Module和Controller，非常简单。

JavaScript 数字类型

发表于 2017-07-30 | 分类于 Node.js ， JavaScript |

JavaScript的数字类型使用的是双精度浮点型（64位二进制），遵循IEEE754标准。

基本知识

IEEE 754标准：标准了定义了单精度浮点数、双精度浮点数，还有扩展精度浮点数，这里只介绍JavaScript使用的双精度浮点数。

双精度浮点数表示

双精度浮点数格式

浮点数使用科学计数法表示，分为符号位s、指数位e、小数位f三部分。64位浮点数中每部分分别占1位、11位、52位。

科学计数法要按规范写法表示，如12.34 = 1.234 * (10^1)。因为规范表示中，小数位中第一位固定是1，所以这可以固定为1，不需要占用52位的小数位，64位浮点数实际上能表示53位小数位。

双精度浮点数值计算

指数位共11位，可以表示0~2047之间的值，但为了表示正负指数，加了一个偏移量bias=1023，指数位实际能表示的指数范围为e-bias=[-1023, 1024]，其中两端的-1023和1024用于表示特殊值。浮点数表示值的公式如下（图片来自IEEE754 浮点数格式 与 Javascript number 的特性）：

特殊值处理

Number中有些特殊值，如 NaN、0、正负无穷等，需要特殊表示。如上图，在e=0和e=2047时，根据f是否等于零，分为四种情况，用于表示四种特殊值。下图是标准里说明的计算方法，其实跟上面那个图是一样的。

其中，e=0并且f!=0时，使用的是非规范表示，小数位没有那个隐含的1，用于表示一些特别小的数。

JavaScript Number里的一些值

Number.MAX_VALUE：能呈现的最大浮点数，即小数位全为1，指数位为1023：

(1.11…1) * 2^1023 = (2^53 - 1) * 2^(-52) * 2^1023 = (2^53 - 1) * 2^971 = 1.798 * 10^308

Number.MIN_VALUE：能呈现的最小浮点数，即小数位最后一位为1，指数为为-1022：

(0.00…1) * 2^(-1022) = 2^(-52) * 2^(-1022) = 2^(-1074) = 5 * 10^(-324)

Number.MAX_SAFE_INTEGER：能安全呈现的最大整数，即小数位全为1，指数为52：

(1.11…1) * 2^52 = 2^53 - 1

Number.MIN_SAFE_INTEGER：能安全呈现的最小整数，就是上面的取负数

Number.EPSILON：机器精度，能表示的比1大的最小数减1：

(1.00…1) -1 = 2^(-52)

Let's Encrypt如何工作的

发表于 2017-07-27 | 分类于 Web ， HTTPS |

原文链接：How It Works，里面图片来自Let’s Encrypt官网。

可以在Let’s Encrypt申请免费HTTPS证书，现在只能申请单个域名的证书（可以申请多个），2018年将支持通配符证书。

Let’s Encrypt证书创建涉及两个过程：域名验证和证书签发。

域名验证

首先要证明用户拥有要设置HTTPS的域名，有多种方式，最简单的是放置一些内容在服务器上，让LE访问，证明自己拥有该域名，如下图所示：

LE告诉网站要如何证明：把一些内容放在域名下，还传递了一个随机字符串。

网站做好上面的要求后，通知LE检查：发送用自己私钥加密的那个随机字符串和自己的公钥给LE，LE用公钥解密，验证那个字符串是否正确。同时访问域名，验证服务器上是否有自己要求放置的内容。这里LE得到了用户的公钥，并用这个公钥标识该用户。

证书签发

上图中，A是用户的公私钥对，S是域名的公私钥对，传送的内容用用户私钥和域名私钥加密了两次。LE接到请求后，用用户的公钥和域名公钥解密里面的内容，里面的内容包含：网站域名和域名公钥。验证成功后返回证书，证书里面包含网站域名和域名公钥。这样用户浏览器访问时就可以获取到域名公钥了。
